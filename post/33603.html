亚虎大百货服务环境搭建

环境需求

运行服务需要的系统环境如下：

服务 版本（版本可不一致，兼容即可）
Centos 7.x
Jdk 1.8
Nginx 1.12.x
Redis 4.x
Mysql 5.7.x
MongoDB 4.x
Docker 18.x
rabbitMQ 3.7.x management（具有可视化面板版本）
Tomcat 9.0.x（war包运行服务，本次环境暂未配置）

Redis搭建

安装要求

参考文档：Linux 安装 Redis

要求：

  • 端口：16379
  • 安装方式：源码安装
  • 自行测试redis-cli客户端是否正常

安装过程

下载解压

  1. 到Redis官网下载最新稳定版：Reids官网
  2. 上传到/usr/local/redis/目录下并解压

解压后的文件在/usr/local/redis/redis-6.2.1目录下

编译安装

进入/usr/local/redis/redis-6.2.1目录，执行编译。编译后会出现redis.conf文件和src目录。

编译成功，进入src目录下，执行make install进行Redis安装。

部署启动

为了方便管理，将Redis文件中的conf配置文件和常用命令移动到统一文件中。

  • 创建bin和etc目录。
  • 将redis.conf文件移动到etc目录下。
  • 进入src目录将其他常用命令移动到bin目录下。

查看移动后的文件

启动测试

redis.conf配置

编辑redis.conf配置文件，根据服务需求配置

  • 注释绑定IP，允许远程连接
  • 允许后台运行
  • 修改模式
  • 配置日志输出位置
  • 设置最大内存,单位：字节【如果内存足够，默认即可】
  • 设置密码
  • 修改端口（非必须😄）

其他的配置，默认即可。

重启Reids

因为是自定义安装的原因，所以重置Redis服务的时候需要指定配置文件redis.conf

redis-cli客户端

将redis-cli路径添加到PATH环境变量中，即可在任意路径下执行redis-cli -a 密码 -p 端口 -h IP地址命令，进入redis-cli客户端。

步骤：

  • 打开/etc/profile文件，编辑路径配置
  • 刷新环境配置：source /etc/profile

redis启动脚本

需求：配置redis脚本，在任意路径下快捷启动、停止redis服务

脚本介绍

  1. 在/etc/init.d/目录下创建脚本 文件 redis_16379

  2. 根据redis各个文件的路径，配置redis_16379脚本文件

  3. 执行脚本、

    • 开启命令：service redis_16379 start

    • 关闭命令：service redis_16379 stop

坑点：

  • redis如果设置有密码，在脚本文件中，必须进行显式定义使用。

  • 使用脚本开启，必须使用脚本关闭，否则会无法在关闭redis的时候系统自动删除/var/run/redis_16379.pid文件。

  • 如果未使用脚本关闭redis服务后，只需要手动删除redis_16379.pid 文件即可再次使用脚本启动。
  • 脚本文件名称可改动，见名知义即可。

数据迁移

方式一：使用mongo自带的mongoexport和mongoimport命令进行数据迁移，原理：先导出集合数据，在导入到mongo数据库中。

方式二：使用mongodump和mongorestore命令进行数据备份和恢复，原理：先备份数据到指定位置，再从指定位置恢复数据

注意：mongodb需要添加到PATH环境变量中

区别：

  • mongodump 比 mongoexport 的速度稍快（使用时没怎么感觉到）
  • mongodump 可以backup整个数据库或者也可以针对每个collection操作，而mongoexport只能对每个collection进行操作
  • mongoexport 输出的JSON比mongodump 的BSON可读性更高

推荐使用：mongoexport和mongoimport进行数据迁移

数据导出

参数说明

导出命令：mongoexport

参数说明：

  • -h, —host ：代表远程连接的数据库地址，默认连接本地Mongo数据库；
  • —port：代表远程连接的数据库的端口，默认连接的远程端口27017；
  • -u,—username：代表连接远程数据库的账号，如果设置数据库的认证，需要指定用户账号；
  • -p,—password：代表连接数据库的账号对应的密码；
  • -d,—db：代表连接的数据库；
  • -c,—collection：代表连接数据库中的集合；
  • -f, —fields：代表集合中的字段，可以根据设置选择导出的字段；
  • —type：代表导出输出的文件类型，包括csv和json文件；
  • -o, —out：代表导出的文件名；
  • -q, —query：代表查询条件；
  • —skip：跳过指定数量的数据；
  • —limit：读取指定数量的数据记录；
  • —sort：对数据进行排序，可以通过参数指定排序的字段，并使用 1 和 -1 来指定排序的方式，其中 1 为升序排列，而-1是用于降序排列,如sort({KEY:1})。

数据备份恢复

数据备份
参数示例

备份命令：mongodump

参数说明：

  • —host <:port>, -h <:port> # 指定备份的主机ip和端口号，默认值localhost:27017
  • —port # 指定端口号 默认27017
  • —username , -u # 指定用户名
  • —password , -p # 指定密码
  • —authenticationDatabase # 指定认证的数据库
  • —authenticationMechanism # 指定认证的算法 ，默认值 SCRAM-SHA-1
  • —db , -d # 指定备份的数据库，未指定的话，备份所有的数据库，但不包含local库
  • —collection , -c # 指定备份的集合，未指定则备份指定库中的所有集合。
  • —query , -q # 指定 json 作为查询条件。来备份我们过滤后的数据。 —queryFile # 指定 json 文档路径，以该文档的内容作为查询条件，来备份我们过滤后的数据。
  • —quit # 通过抑制 MongoDB的复制，连接等活动，来实现备份。
  • —gzip # 开启压缩，3.2版本后可以使用，输出为文件的话会带有后缀.gz
  • —out , -o # 输出的目录路径（如果指定集合的备份，默认会输出的目录格式：当前目录/数据库名/集合名.bson）
命令示例
数据恢复
参数示例

恢复命令：mongorestore

参数说明：

  • —host <:port>, -h <:port> # 指定恢复的主机ip和端口号，默认值localhost:27017
  • —port # 指定端口号 默认27017
  • —username , -u # 指定用户名
  • —password , -p # 指定密码
  • —authenticationDatabase # 指定认证的数据库
  • —authenticationMechanism # 指定认证的算法 ，默认值 SCRAM-SHA-1
  • —objcheck # 开启验证，验证还原操作，确保没有无效的文档插入数据库。会有较小的性能影响
  • —oplogReplay # 恢复备份数据并将 mongodump 执行期间的操作(记录在导出的日志)恢复。
  • —oplogLimit # 指定恢复
  • —oplogFile # 指定 Oplog 路径
  • —keepIndexVersion # 阻止mongorestore在还原过程中将索引升级到最新版本。
  • —restoreDbUsersAndRoles # 还原指定的数据库用户和角色。
  • —maintainInsertionOrder # 默认值为False,如果为 True,mongorestore 将按照输入源的文档顺序插入，否则是 随机执行插入。
  • —numParallelCollections int, -j int # 指定并行恢复的集合数。
  • —numInsertionWorkersPerCollection int # 默认值为 1，指定每个集合恢复的并发数，大数据量导入增加该值可提高 恢复速度。
  • —gzip # 从压缩文档中 恢复。
  • —archive # 从归档文件中恢复。
  • —dir # 指定还原数据储存目录。

nginx配置

upstream服务流

当配置upstream流之后，nginx自动触发负载均衡。

nginx负载均衡策略主要有3种：

  • 轮询策略： nginx默认的负载均衡策略
  • 权重策略：在配置upstream服务流时，手动配置每个服务地址的权重
  • ip_hash策略：每个请求按照访问 ip 的 hash 结果分配，这样只要 ip 访问过并被分配过服务器之后，以后再次方位，分配的都将是同一个服务器

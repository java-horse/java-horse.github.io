SpringBoot结合Redis的bitmap实现签到功能和布隆过滤器

每月签到

签到功能需求：

  • 记录每个用户每月的签到情况（签到或者未签到、签到总天数、总连续到天数等），这里，基于用户ID（userId）和每月日期（yyyyMM）作为key，结合Redis中的bitmap数据结构，实现用户每月签到情况记录的功能。

本文结合仅仅实际代码片段，进行签到功能复现（重在核心逻辑，因为每个签到需求可能都不一样）

片段讲解

公共代码片段

命令代码片段

setbit

redis命令：SETBIT key offset value

命令含义：将名为key的第offset位的值置为 1，（offset偏移量从 0 开始）

逻辑理解：把每月的每天是当月的第多少天计算出来，作为 offset 偏移量的值。如果当天签到，就把对应位的值置为 1，否则默认值就为 0。后面可根据1和0的值即可得出用户每月的签到情况。

getbit

redis命令：GETBIT key offset

命令含义：获取第 offset 位的对应值

逻辑理解：使用setbit命令进行签到后，可以使用此命令获取 用户当前的签到状态（已签到：返回true，未签到：返回false）

bitcount

redis命令：BITCOUNT key start end

命令含义：根据key获取对应值的start 到 end 范围内 1 的个数 （start ,end不写 默认获取全部范围，或者 0 -1 也表示全部范围）

逻辑理解：使用此命令获取用户当月的总签到数

注意：redisTemplate并没有提供直接的方法来调用bitcount，可以通过redisTemplate.execute来执行bitcount方法

bitfield

redis命令：BITFIELD key get [u/i]offset start

命令含义：

  • u：代表无符号位
  • i：代表有符号位
  • start：代表从哪一位开始获取，0代表从第一位开始获取
  • offset：代表共获取多少位的值

逻辑理解：使用 bitfield 获取用户签到的总值，再根据其他的逻辑进行计算获取 此用户每月的连续签到情况。

根据和上面类似的逻辑，获取用户当月每天的签到情况

bitop

redis命令：bitop [add/or/xop/not] destKey keys…

命令含义：

  • and：对一个或多个 key 求 逻辑并，并将结果保存到 destkey
  • or：对一个或多个 key 求 逻辑或，并将结果保存到 destkey
  • xor：对一个或多个 key 求 逻辑异或，并将结果保存到 destkey
  • not：对给定 key 求逻辑非，并将结果保存到 destkey
  • 注意：除 not 操作外，其余操作都支持一次输入一个或者多个key

逻辑理解：使用 用户id 去匹配查询由自己签到生成的 bitmap值，通过bitop命令统计此用户的签到总天数。

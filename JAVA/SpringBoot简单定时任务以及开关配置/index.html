注解简介

利用SpringBoot中提供的@ConditionOnProperty注解，实现项目启动过程中是否加载注解所在的这个配置项。

yml配置属性

注解解析

@ConditionOnProperty注解基本作用：

通过属性name以及havingValue来实现的，其中name用来从application.properties中读取某个属性值。

  • 如果该值为空，则返回false;
  • 如果值不为空，则将该值与havingValue指定的值进行比较，如果一样则返回true;否则返回false。
  • 如果返回值为false，则该configuration不生效；为true则生效。

注意：prefix 表示某些属性值的前缀

缺点：每次更新定时任务,只能手动停止服务,更新cron表达式,再打包,重新启动服务,只适用于更新不频繁的定时任务。

SchedulingConfigurer接口实现

缺点：虽然实现了cron表达式的动态配置,但是每次配置之后,表达式生效的时间是下次执行此任务的时候,也就是说,动态配置不能实时生效,简单的定时任务可以这样使用,即可以满足需求,但是稍微复杂的,实时性比较高的,建议使用Quartz, xxl-job等任务框架

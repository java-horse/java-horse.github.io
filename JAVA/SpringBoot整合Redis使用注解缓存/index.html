SpringBoot整合Redis使用注解缓存

发表于2021-05-21 22:00:52|更新于2022-01-08 11:38:05|开发日常
|字数总计:1k|阅读时长:3分钟|阅读量:

缓存原理

CacheManager === Cache 缓存组件会实际项目中引入的缓存组件，进行数据的缓存。

我们可以对redisCacheManager缓存管理器进行一定的配置

  1. 引入redis的starter，容器中保存的是RedisCacheManager
  2. RedisCacheManager 帮我们创建RedisCache 来作为缓存组件，RedisCache通过操作redis来缓存数据
  3. 默认保存数据 K -V 都是通过序列化来保存的

注意事项：

  • 需要被序列化缓存的实例，必须实现Serializater选项

Redis缓存配置

核心注解简析

@EnableCaching

作用在项目启动类或者某个配置类上使用此注解，表示：允许使用注解的方式进行缓存操作

注意：此【类】必须是Spring容器中的，否则注解不会生效

@Cacheable

作用目标：类或者方法上

缓存效果：在目标方法执行前，会根据key先去缓存中查询看是否有数据，有就直接
返回缓存中的key对应的value值。不再执行目标方法；无则执行目标方法，并将方法的返回值
作为value，并以键值对的形式存入缓存

@CachePut

作用目标：类或者方法上

缓存效果：触发缓存更新操作，添加该注解后总是会执行方法体，并且使用返回的结果更新缓存

@CacheEvict

作用目标：类或者方法上

缓存效果：在执行完目标方法后，清除缓存中对应key的数据(如果缓存中有对应key的数据缓存的话)

@CacheConfig

作用目标：一般作用与类上

效果：@Cacheable、@CacheEvict、@CachePut这三个注解的cacheNames属性是必 填项(或value属性是必填项,因为value属性是cacheNames的别名属性)；如果上述三种注解都用的是同一个cacheNames的话，那么在每此都写cacheNames的话，就会显得麻烦。如将@CacheConfig注解就是来配置一些公共属性(如：cacheNames，keyGenerator等)的值。

主要属性：

参数 含义
value 缓存的名称，必须指定一个，相当于nacos配置中的命名空间（不同的value中，可以有相同的key）
key 缓存的key值，分为三类：默认的、keyGenerator生成的、主动指定的
condition 在激活注解功能前，进行condition验证，如果condition结果为true，则表明验证通过，缓存注解生效；否则缓存注解不生效
unless 是否令注解（在方法执行后的功能）不生效；若unless的结果为true，则(方法执行后的功能)不生效；若unless的结果为false，则(方法执行后的)功能生效

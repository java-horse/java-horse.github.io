检索查询命令

vi/vim检索

应用场景: 日志查询,精准匹配

  • 使用 / 搜索

命令模式下，输入/关键字 开始自上向下搜索。n跳到下一个，N跳到上一个。3n跳到向下第三个
> \c 表示不区分大小写 例如: /error\c
> \C 表示区分大小写 例如: /error\C

  • 使用 ? 搜索

和 / 类似，但是方向相反
> \c 表示不区分大小写
> \C 表示区分大小写

  • 可以先设置搜索规则,再使用上面的符号进行搜索进行搜索

  • 是否区分大小写

    • 不区分大小写：:set ignorecase
    • 区分大小写：:set noignorecase

  • 是否实时搜索

    • 开启实时搜索：:set incsearch
    • 开启只能搜索：:set smartcase

  • 是否显示行号

    • 显示行号：:set number 或者 :set nu
    • 取消显示：:set nonumber 或者 :set nonu

  • 是否高亮显示

    • 永久去除搜索高亮显示：:set nohls
    • 恢复搜索高亮：:set hls
    • 暂时取消高亮（下次搜索时再次显示）：:noh

vim常用内置命令

grep命令

  • grep简单匹配搜索

grep -n ‘error’ todo.txt [搜索todo.txt文件中匹配’error’的所在行和行号]

grep –e “正则表达式” 文件名 [从文件内容查找与正则表达式匹配的行：]

grep –i “被查找的字符串” 文件名 [查找时不区分大小写：]

grep -c “被查找的字符串” 文件名 [查找匹配的行数：]

grep -n ‘error’ *.log [在当前目录下查找所有扩展名为 .log 的文本文件，并找出包含 “error” 的行]

ls | grep ‘^app’ 过滤文件中以 app 开头的文件[查询很快]

  • grep命令的 与 或者 或 操作

grep -E ‘word1|word2|word3’ filename [ 查询 filename 文件中 包含 Word1，word2，word3 中任意一个关键词的行 ]

简便写法：

  • egrep ‘word1|word2|word3’ filename
  • awk ‘/word1|word2|word3/‘ filename

grep ‘word1’ filename | grep ‘word2’ | grep ‘word3’ [ 查询 filename 文件中 同时包含 word1，word2，word3 关键词的行，（也就是多级管道符过滤操作而已！）]

find命令

  • 常用匹配搜索

find /home -name *.jar [在/home目录下根据文件名进行匹配查询]

find /home -iname horse-vue-blog.jar [在/home目录下根据文件名进行匹配查询(忽略大小写)]

find . -ctime -20 [查询当前目录及其子目录下所有最近20天内更新过的文件]

cat,more,head,tail等命令

  • 常用操作

cat -nbs todo.txt [查看文件]

more -s todo.txt [逐页阅读，而最基本的指令就是按空白键（space）就往下一页显示，按 b 键就会往回（back）一页显示]

head -n 20 todo.txt [查看文件的开头20行部分内容,默认为查看开头10行]

tail -f todo.log [循环读取todo.log文件的内容]

tail -n 20 todo.log [查看todo.log文件末尾20行内容]

tail -f -n 200 todo.log | perl -pe ‘s/(关键词)/\e[1;31m$1\e[0m/g’ [高亮关键词]

less命令

命令简介：Linux中查看文件内容的强大工具

命令功能：less 与 more 类似，但使用 less 可以随意浏览文件，而 more 仅能向前移动，却不能向后移动，而且 less 在查看之前不会加载整个文件

命令参数：

  • -b <缓冲区大小> 设置缓冲区的大小

  • -e 当文件显示结束后，自动离开

  • -f 强迫打开特殊文件，例如外围设备代号、目录和二进制文件

  • -g 只标志最后搜索的关键词

  • -i 忽略搜索时的大小写

  • -m 显示类似more命令的百分比

  • -N 显示每行的行号

  • -o <文件名> 将less 输出的内容在指定文件中保存起来

  • -Q 不使用警告音

  • -s 显示连续空行为一行

  • -S 行过长时间将超出部分舍弃

导航参数：

  • G - 移动到最后一行（shift + g）

  • g - 移动到第一行

  • q / ZZ - 退出 less 命令

whereis命令

命令简介：和find相比，whereis查找的速度非常快，这是因为linux系统会将 系统内的所有文件都记录在一个数据库文件中，当使用whereis和下面即将介绍的locate时，会从数据库中查找数据，而不是像find命令那样，通过遍历硬盘来查找，效率自然会很高。

但是该数据库文件并不是实时更新，默认情况下时一星期更新一次，因此，我们在用whereis和locate 查找文件时，有时会找到已经被删除的数据，或者刚刚建立文件，却无法查找到，原因就是因为数据库文件没有被更新。

firewall防火墙

查看防火墙状态

查看防火墙状态

  • firewall-cmd --state
  • systemctl status firewalld

防火墙的开启、重启、关闭

防火墙的开启、重启、关闭等操作（具体效果不再做演示，记得就好）

  • 开启
    • systemctl start firewalld
    • service firewalld start
  • 重启
    • systemctl restart firewalld
    • service firewalld restart
  • 关闭
    • systemctl stop firewalld
    • service firewalld stop

查看防火墙规则

查看防火墙全部规则：firewall-cmd --list-all

防火墙端口常用操作

查看防火墙所有已开启的端口：firewall-cmd --list-ports

添加新的端口：firewall-cmd --zone=public --add-port=12701/tcp --permanent

添加连续的端口：firewall-cmd --zone=public --add-port=8080-8848/tcp --permanent

关闭端口：firewall-cmd --zone=public --remove-port=12701/tcp --permanent

查询某个端口是否开放：firewall-cmd --query-port=6379/tcp

注意：对于端口的操作，必须重启防火墙才能生效。

参数解释：

  • firwall-cmd：是Linux提供的操作firewall的一个工具
  • –permanent：表示设置为持久，没有此参数重启后失效
  • –add-port：标识添加的端口
  • –remove-port：标识要移除的端口
  • –query-port：标识要查询的端口
